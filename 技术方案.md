# AgentLens MVP 开发提示词

## 项目目标

构建一个本地运行的 AI Agent Context Inspector。

产品定位：

> The DevTools for AI Agent Context

核心价值：

让开发者看见 AI Agent 实际拿到了什么上下文，而不是猜测。

支持：

* Claude Code（第一阶段唯一支持）
* 本地 JSONL 数据
* 本地 Web UI

暂不考虑：

* 用户系统
* 云同步
* 数据库
* 多人协作
* MCP 管理
* AI 总结

---

# 用户核心问题

用户打开 AgentLens 后必须在 3 秒内回答：

### 1

```text
CLAUDE.md 加载了吗？
```

### 2

```text
Skill 加载了吗？
```

### 3

```text
Memory 注入了吗？
```

### 4

```text
MCP 工具注册了吗？
```

### 5

```text
最终发送给模型的 Prompt 是什么？
```

### 6

```text
这一轮和上一轮发生了什么变化？
```

仅做四个页面。


# 页面1：Overview

首页。

回答：

```text
AI 到底拿到了什么？
```

布局：

```text
Loaded Context

CLAUDE.md
Memory
Skills
Rules
Steering
MCP
```

每个项目显示：

```text
名称

状态

Token数

来源
```

例如：

```text
✓ Architecture Skill

4231 Tokens

.skills/architecture.md
```

---

下面显示：

```text
Context Usage
```

内容：

```text
History

Skills

Rules

Memory

Tools
```

显示占比。

使用横向 Bar。

不要饼图。

---

下面显示：

```text
Warnings
```

例如：

```text
History exceeds 60%

14321 tokens truncated
```

# 页面2：Prompt Inspector

这是产品核心。

不要展示原始 JSON。

目标：

让用户查看最终 Prompt 组成。

布局：

左侧：

```text
System

Identity
Rules
CLAUDE.md
Memory
Skills
MCP Metadata
```

中间：

显示当前选中内容。

例如：

```text
Architecture Skill

Source:
.skills/architecture.md

Tokens:
4231

Content:
...
```

右侧：

显示元数据。

```text
Loaded

Priority

Token Count

Source Path
```

---

要求：

所有内容支持点击。

点击树节点。

更新中间内容区域。

---

# 页面3：Context Diff

对比：

```text

vs

上一请求
```

新增：

```text
+ architecture skill
```

删除：

```text
- frontend skill
```

变化：

```text

```

截断：

```text
14321 tokens removed
```

支持：

```text
Added

Removed

Modified
```

分类。

# 产品原则

不要做监控平台。

不要做 Grafana。

不要做 Token Dashboard。

不要做 AI 总结。

不要做复杂图表。

目标只有一个：

> 让开发者看见 Agent 实际拿到了什么上下文，以及上下文是如何变化的。

如果用户第一次打开后能回答：

```text
原来 Architecture Skill 没加载

原来 History 吃掉了 60% Context

原来 Memory 被截断了
```

则 MVP 成功。
